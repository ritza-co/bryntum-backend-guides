# Bryntum Backend guides code

Code for Bryntum docs backend guides:

- [Bryntum Calendar Backend Guides](https://bryntum.com/products/calendar/docs/api/integration-backends)
- [Bryntum Gantt Backend Guides](https://bryntum.com/products/gantt/docs/api/integration-backends)
- [Bryntum Grid Backend Guides](https://bryntum.com/products/grid/docs/api/integration-backends)
- [Bryntum Scheduler Backend Guides](https://bryntum.com/products/scheduler/docs/api/integration-backends)
- [Bryntum Scheduler Pro Backend Guides](https://bryntum.com/products/schedulerpro/docs/api/integration-backends)
- [Bryntum Task Board Backend Guides](https://bryntum.com/products/taskboard/docs/api/integration-backends)

The structure of this repo is as follows:

```
├── backend/
│   └── dotnet/
│       ├── sqlite-calendar/
│       ├── sqlite-gantt/
│       ├── sqlite-grid/
│       ├── sqlite-scheduler/
│       ├── sqlite-schedulerpro/
│       └── sqlite-taskboard/
│
└── frontend/
    └── vanilla-js/
        ├── calendar/
        ├── gantt/
        ├── grid/
        ├── scheduler/
        ├── schedulerpro/
        └── taskboard/
``` 
                                                                      
These examples use vanilla JS, if you use a frontend framework, take a look at the React, Angular, and Vue quickstart guides in our docs to get started.
